
Информация о конфигурации Hibernate определена в файле hibernate.cfg.xml 

Свойства, определяющие информацию о соединении JDBC.
connection.driver_class - класс драйвера jdbc
connection.url - url bd
connection.username - имя пользователя
connection.password - пароль пользователя 
<property/> 
connection.pool_size - размер пула соединений.
dialect - свойство указывает конкретный вариант SQL, с которым будет взаимодействовать Hibernate.
hbm2ddl.auto - свойство позволяет автоматически создавать схемы баз данных непосредственно в базе данных.

Требования к сущнисти в java-классах
	- Этот класс использует стандартные Соглашения об именовании JavaBean для методов getter и setter свойств, а также private видимость для полей.
	- Конструктор без аргументов, который также является соглашением JavaBean, является требованием для всех постоянных классов. Hibernate должен создавать объекты для вас, используя отражение Java. Конструктор может быть приватным. Однако пакетная или общедоступная видимость необходима для создания прокси-сервера во время выполнения и эффективного извлечения данных без инструментария байт-кода.



The mapping file
	
	Маппинг класса выполняется по шаблону <varname>class</varname>
		Пример сопоставления классов таблицам ИБ, где экземпляры класса Event теперь сопоставляются со строками в таблице EVENTS базы данных.
			<class name="Event" table="EVENTS">
				...
			</class>
		Где
			name - задает полное имя класса, определяемого как сущность. (объединяется с атрибутом package в <hibernate-mapping/>)
			table - объявляет таблицу базы данных, которая содержит данные для этого объекта.
	
		Пример сопоставления по идентификатору
			<id name="id" column="EVENT_ID">
				...
			</id>
		Hibernate использует свойство <id/>, чтобы однозначно идентифицировать строки в таблице.
